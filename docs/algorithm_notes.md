# 算法说明

## 状态表示

`cpp_2048_agent.cpp` 使用 64-bit bitboard 表示棋盘。每个格子 4 bit，存储的是 tile 的指数而不是显示值。这样 16 个格子刚好放进一个 `uint64_t`。

例如：

```text
显示值: 0 2 4 8
指数值: 0 1 2 3
```

优点：

- 盘面复制成本低。
- 可以直接作为哈希表键。
- 行、列移动能用位运算快速抽取。

## 行表预处理

一行有 4 个格子，每格 4 bit，所以一行共有 `2^16` 种编码。程序启动时预计算：

- `row_left`
- `row_right`
- `row_left_reward`
- `row_right_reward`
- `row_empty`
- `row_can_move`
- `row_score`

上下移动通过抽取列并复用行表完成。

## Expectimax

2048 不是完全确定性游戏，因为每次移动后会随机生成新块。搜索分两层：

```text
max_node: 玩家选择方向
chance_node: 环境生成 2 或 4
```

`chance_node` 按概率计算期望：

```text
E = Σ cell_probability * (0.9 * V(spawn 2) + 0.1 * V(spawn 4))
```

搜索优化包括：

- 根据最大块和空格数动态调整深度。
- 在 chance 层只选择高优先级空格。
- 使用 transposition table 避免重复搜索。
- 使用概率阈值和时间阈值提前返回静态估值。

## 启发式

搜索深度有限时，叶子节点必须通过估值函数判断优劣。项目使用的主要特征：

- 空格越多越好。
- 行列越单调越好。
- 相邻可合并块越多越好。
- 最大块在角落更好。
- 大块链不被破坏更好。
- 产生 4096、8192、16384 等里程碑时给予额外奖励。

这类启发式不是严格数学最优，但能把人类经验编码进搜索，在有限时间内得到稳定决策。

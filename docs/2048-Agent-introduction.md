# 2048-Agent 项目介绍

由于本人非常喜欢玩 2048，因此本次数算 B 大作业选择完成一个有关 2048 的 Agent 项目，并尝试从算法运行结果中解读一些 2048 的技巧与布局思路。

GitHub 仓库链接：[https://github.com/jerry2124679-cpu/2048-Agent](https://github.com/jerry2124679-cpu/2048-Agent)

本项目以经典游戏 2048 为载体，围绕“随机环境下的智能决策”展开算法实践。项目将每一步移动抽象为搜索问题：程序需要在有限时间内比较上下左右四种选择，并估计移动后随机生成新方块所带来的后续影响。为此，项目综合运用了状态压缩、递归搜索、哈希缓存、启发式评估和复杂度优化等知识，实现了一个能够自动游玩、可视化演示并记录复盘过程的 2048 AI 智能体。通过该项目，可以直观看到数据结构设计和算法策略如何影响程序性能，也能体现从理论算法到完整工程实现的实践过程。更多介绍详见 GitHub 仓库的 [README.md](https://github.com/jerry2124679-cpu/2048-Agent)。

2048 的基础策略通常是层叠法，也就是按照一层一层堆砌的方式完成布局。这种方法简单、直观，适合在前中期保持最大块稳定增长，通常在合成 4096 之前比较常用。

但这种策略也有明显局限：它通常要求玩家长期避免某一个方向的移动，一旦被迫反向操作，就可能破坏最大块所在的角落结构，导致高价值数字链断裂。因此，在合成 4096 之后，需要考虑更适合后期局面的空间划分方法，例如 628 和 63L。

628 和 63L 本质上都是 2048 后期的“分区经营”策略。628 可以理解为：先把 6 个相对较大的数稳定地摆成一个 `2 x 3` 的核心区，这个区域负责保存主力数字链；再利用剩下的 2 个格子补充中间过渡数字，使大块之间保持可衔接关系；最后把剩下的 `2 x 4` 区域当作工作区，用来生成小数字、合并小数字、向主链输送材料。63L 则更偏向另一种空间划分：同样保留 6 个较大的核心数字，再额外用 3 个格子填入次级数字，把大块链包成更稳的结构，而剩余的 L 形区域作为工作区。前者的特点是工作区更规整、更宽，适合持续生产和整理小块；后者的特点是核心区更封闭、更抗扰动，适合在后期棋盘压力较大时保护高价值数字。

在本项目中，Agent 并没有显式学习这些人类定式，也没有被写入“必须摆成 628 或 63L”的规则，而是通过 Expectimax 搜索与启发式估值函数，在大量候选局面中自动选择更稳定的盘面结构。因此，628 和 63L 的出现可以看作算法目标与人类经验策略之间的一种重合。

算法之所以会倾向于形成这两类布局，首先是因为 6 个大数集中成 `2 x 3` 区域，可以让最大块和次大块保持在角落或边缘附近，减少它们被随机生成的小数字切断的风险。对于 2048 来说，大块一旦散开，后续很难重新合并；而大块集中摆放时，移动方向更稳定，合并路径更清楚，算法评估时自然会给出更高分。

其次，这两种策略都能维持较好的单调性。2048 的理想结构通常不是把大数随意堆在一起，而是让数字按照从大到小的顺序沿边缘或转角递减。628 的 `2 x 3` 主区和 63L 的 L 形主链，都容易形成类似“最大块在角落，旁边依次是次大块、更小块”的梯度结构。这样的结构在上、下、左、右移动时不容易被打乱，也更容易把新合成的数字顺着路径送回主链。算法中的单调性评分、角落保护和大块链保护都会共同偏向这种形态。

再次，628 和 63L 都保留了独立工作区，这是它们比单纯“把大数全部塞满一边”更稳的地方。2048 的随机生成机制决定了棋盘必须有足够空间处理新出现的 `2` 和 `4`。如果大块区占得太满，小块没有地方合并，棋盘很快会被杂乱数字堵死。628 保留 `2 x 4` 工作区，空间比较开阔，适合反复进行小数字合成；63L 虽然工作区形状不如矩形规整，但它把核心大块保护得更严密，适合后期在空间紧张时减少主链被破坏的概率。算法在搜索未来分支时，会发现这类局面既能保留空格，又能维持合并潜力，因此期望收益较高。

最后，从 Expectimax 的角度看，随机生成的新块会带来很多不确定分支，而 628 和 63L 能降低这种不确定性带来的损失。因为大块区相对固定，随机新块大多会落在工作区，算法只需要在工作区内消化小块，再把合成结果逐步送入主链。这样的局面在许多随机分支下都不会立刻崩盘，平均期望值自然更高。换句话说，算法不是在追求某一个瞬间的最高得分，而是在追求“多数随机情况都还能继续玩下去”的稳定结构；628 和 63L 正好提供了这种稳定性，所以会成为搜索和估值共同作用下反复出现的布局倾向。

因此，算法会在后期自然倾向于 628 和 63L 的布局，这对人类玩家也具有一定启发意义：好的 2048 策略不仅要追求当前合并，更要保护高价值数字链、保留工作区，并尽量降低随机生成带来的破坏。

为了让 Agent 能够合成更大的数，接下来可以考虑进行残局蒸馏，即针对固定的残局形态构建表格，并让 Agent 在这些典型后期结构上进行更精细的定式训练。这样可以进一步提升后期决策的稳定性，使算法在接近高阶方块时减少失误。
